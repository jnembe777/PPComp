02 — Fonctionnalités
Six capacités impossibles avec les codecs classiques
Chaque fonctionnalité découle naturellement du paradigme processus ponctuel. Rien n'est ajouté — tout est structurel.
🎨

Chroma VHR

8 à 32 bits de profondeur couleur sans impact sur la compression. Le coût dépend de la palette, pas de la profondeur. Surcoût 8→24 bits : 3%.

🔍

Zoom Natif O(1)

Accès aléatoire à n'importe quel bloc en temps constant via offset table. Zoom sans re-buffering. Overhead : 0.7%.

📊

LOD Variable

3 niveaux de détail : LOD 0 (0.001 bpp), LOD 1 (1 bpp), LOD 2 (4 bpp). Le décodeur investit uniquement sur les zones actives.

🏷️

Requêtes Sémantiques

Interroger la vidéo sans la décoder. SQL-like sur le bitstream compressé en O(log durée). Overhead tags : 5%.

🧊

Vidéo 3D Native

Blocs cubiques 6 faces avec timestamps partagés. Navigation directionnelle. Coût : 1.5-2× le 2D.

🏔️

Pyramide Progressive

Streaming N₀→N₅ : le résumé arrive en premier, le détail suit. Liaison interrompue = image exploitable.

04 — Requêtes Sémantiques
Interroger la vidéo comme une base de données
SQL-like directement sur le flux compressé. Pas de décodage. Pas de pipeline IA externe. Les métadonnées sont dans le bitstream.
SELECT time, location, tag
FROM video_stream
WHERE tag = 'véhicule' AND confidence > 0.8
ORDER BY time ASC
