PPV Browser
Le Web Temporel

Un navigateur natif pour les flux de processus ponctuels. HTML augmenté de primitives temporelles, sémantiques et spatiales. Le web ne montre plus des images — il navigue dans des événements.

01 — La Vision
HTML a été conçu pour des documents.
PPML est conçu pour des flux temporels.
Le langage PPML (Point Process Markup Language) étend HTML5 avec des primitives natives pour les processus ponctuels marqués. Chaque élément du DOM est un flux d'événements horodatés — pas une valeur statique. Le navigateur PPV interprète ces flux nativement, sans JavaScript, sans plugin, sans décodeur externe.

HTML5 — Statique

📷 <img> Image fixe
🎥 <video> Flux linéaire
🔲 <canvas> Dessin impératif
📝 <p> Texte statique
🔗 <a href> Lien spatial
📊 <table> Données tabulaires

02 — Les 8 Primitives PPML
Chaque balise est un processus ponctuel
Les primitives PPML ne sont pas des extensions cosmétiques. Elles sont des objets mathématiques — chaque élément du DOM est un processus ponctuel marqué avec intensité, palette et adresse combinatoire.
<pp-stream>
Flux d'événements temporels. Remplace <video>. Le navigateur décode nativement le format .ppv — pas de codec externe, pas de <source> avec type MIME.
<pp-stream src="flux.ppv" lod="auto" framework="marked" lossless> </pp-stream>
<pp-zoom>
Zone de zoom natif O(1). L'utilisateur zoome avec la molette — le navigateur lit l'offset table et décode uniquement les blocs visibles. Zéro re-buffering.
<pp-zoom target="#cam1" min-lod="0" max-lod="2" access="O(1)"> </pp-zoom>
<pp-pyramid>
Streaming pyramidal progressif N₀→N₅. Le navigateur affiche d'abord le résumé (20 bits), puis enrichit progressivement jusqu'au détail pixel.
<pp-pyramid src="mars.ppv" levels="N0 N2 N3 N5" priority="progressive"> <!-- N0 arrive en 0.1s, N5 en 30s --> </pp-pyramid>
<pp-query>
Requête sémantique SQL-like directement dans le DOM. Le résultat est un flux filtré d'événements — sans décoder la vidéo complète.
<pp-query source="#cam1" where="tag='vehicle' AND conf>0.8" select="time, location, thumbnail" render="timeline"> </pp-query>
<pp-cube>
Vue 3D native — cube 6 faces avec navigation directionnelle. Chaque face est un <pp-stream> avec timestamps partagés. LOD adaptatif par face.
<pp-cube src="scene360.ppv" fov="90" front-lod="2" back-lod="0" navigate="gyroscope"> </pp-cube>
<pp-palette>
Palette de couleurs dynamique — LUT appliquée instantanément sans re-encodage. Chroma VHR 8 à 32 bits. Le grading couleur est une propriété CSS.
<pp-palette target="#cam1" depth="24bit" lut="dolby-vision.cube" live> </pp-palette>
<pp-event>
Liaison DOM ↔ événement temporel. Quand un saut PP se produit, le navigateur émet un événement DOM natif. JavaScript peut écouter les changements de pixel.
<pp-event source="#cam1" on-jump="handleMotion" filter="magnitude>0.5" throttle="100ms"> </pp-event>
<pp-intensity>
Visualisation de la fonction d'intensité α̂(t). Affiche la densité d'activité en temps réel — courbe, heatmap ou histogramme intégré au DOM.
<pp-intensity source="#cam1" family="histogram-K8" render="heatmap" overlay> </pp-intensity>

04 — Exemple Complet
Une page PPML de surveillance
surveillance-dashboard.ppml
<!-- PPML 1.0 — Point Process Markup Language --> <!DOCTYPE ppml> <ppml version="1.0" xmlns:pp="https://ppml.org/2026"> <head> <title>Surveillance Parking Nord</title> <meta pp-codec="ppv/2.0" pp-lossless="required"/> <style> pp-stream { width: 100%; border-radius: 8px; } pp-stream::lod(0) { filter: blur(2px); opacity: 0.7; } pp-stream::lod(2) { filter: none; } pp-query { display: block; max-height: 300px; overflow-y: auto; } pp-intensity { position: absolute; mix-blend-mode: screen; } </style> </head> <body> <!-- Flux principal avec zoom natif --> <pp-stream id="cam1" src="ppv://cam.local/nord.ppv" framework="auto" lod="auto" lossless> <!-- Overlay d'intensité --> <pp-intensity family="histogram-K8" render="heatmap" overlay/> <!-- Zone de zoom --> <pp-zoom min-lod="0" max-lod="2" gesture="pinch wheel"/> </pp-stream> <!-- Requête sémantique interactive --> <pp-query source="#cam1" where="tag='vehicle' AND confidence > 0.8" select="time, block, tag, confidence" render="table" on-click="seek"> </pp-query> <!-- Écoute des événements PP --> <pp-event source="#cam1" on-jump="logMotion" filter="magnitude > 0.5"/> <script> function logMotion(event) { // event.detail = { time, block, mark, intensity } console.log(`Mouvement détecté: ${event.detail.tag}`); } </script> </body> </ppml>
05 — Protocole ppv://
Un protocole natif pour les flux temporels
Le navigateur PPV résout les URI ppv:// nativement. Le protocole transporte des GOPs compressés avec seek table intégrée — le navigateur peut sauter directement au bloc demandé sans parser l'intégralité du flux. C'est le STV-URI du docA en action.
Schéma URI ppv://
ppv://host/path.ppv#t=120&b=5,3&lod=2&tag=vehicle Exemples : ppv://cam.local/nord.ppv // Flux complet ppv://cam.local/nord.ppv#t=120 // Frame 120 ppv://cam.local/nord.ppv#b=5,3&lod=2 // Bloc (5,3) en LOD 2 ppv://cam.local/nord.ppv#tag=vehicle&t=0-300 // Véhicules, 10s ppv://cam.local/nord.ppv#face=front&fov=90 // Face avant, 90° ppv://mars.space/sol-847.ppv#level=N2 // Preview basse rés
